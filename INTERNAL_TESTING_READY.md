## 当前实现范围

### 四个主 Tab

- **今日**: 过去 / 今天 / 明天，日历式时间轴，空白时间折叠，实验以时间段事件展示。
- **Protocol**: 本地方法资产库，支持变量、试剂/步骤、来源记录、收藏/最近、编辑和一致性检查。
- **工具**: 质量浓度、液体稀释、百分比浓度、自定义公式、计算历史和缓冲液模板。
- **我的**: 个人资料、偏好设置、Pro 占位、本地库存入口、库存交易记录、演示数据重置。

### 当前重点功能

- 从今日页创建今天/明天的实验，可选择 Protocol、手动实验或顺延占位。
- 今日实验卡片按起止时间嵌入时间轴，类似 Apple 日历事件。
- 实验步骤支持完成勾选，完成后标题和内容划线。
- 详情页用于编辑实验信息和步骤参数。
- Bench Mode 用于当前实验的大屏执行，可与详情页互相切换。
- Data Card 只展示结果图/预览、实验条件、元数据、水印和分享/保存/复制操作，不包含导师汇报概要。

## 本地持久化范围

当前使用 `UserDefaults`、`@AppStorage` 和 `Codable` JSON 数据保存状态。

已在代码层确认的持久化键包括：

- `importedLabRuns`: 今天新增实验
- `tomorrowLabRuns`: 明天计划
- `pastExperimentDays`: 过去实验记录
- `completedStepIDs`: 已完成步骤
- `activeLabTimers`: 活跃计时器
- `inventoryItems`: 库存项
- `inventoryTransactions`: 库存交易记录
- `userProjects`: 用户项目
- `calculationHistory`: 计算历史
- `customBufferTemplates`: 自定义缓冲液模板
- `savedCustomFormulas`: 自定义公式
- `savedProtocols`: 保存的 Protocol
- `protocolFavoriteIDs`: 收藏 Protocol
- `protocolRecentIDs`: 最近 Protocol
- `profileDisplayName` / `profileLabName`: 我的页面资料
- `preferences*`: 用户偏好
- `customWorkflowAreas` / `inventoryCustomCategories` / `customUnits`: 自定义分类和单位

### 持久化测试方法

1. 在模拟器启动 App。
2. 新增一个今天实验和一个明天实验。
3. 勾选至少两个步骤。
4. 启动一个步骤计时器。
5. 修改一个库存数量并新增一条库存记录。
6. 保存一次计算历史或自定义公式。
7. 修改“我的”里的偏好。
8. 彻底退出 App 后重新打开。
9. 检查上述状态是否仍然存在。

注意：删除 App、清空模拟器数据或执行演示数据重置会清除对应本地数据。

- 包内包含 Xcode 工程、源码、脚本、README 和验收文档

## 已知限制

- v1 没有后端和账号，因此不支持跨设备同步、云备份、团队共享或远程恢复。
- 目前主要是 SwiftUI 原型级持久化，后续产品化建议从 `UserDefaults` 迁移核心业务数据到 SwiftData。
- AI 语音排程、AI 文献/说明书提取、AI 导师助手均为 Pro 后置功能，当前不接入 AI SDK。
- Live Activities / Dynamic Island / 锁屏通知仍需真实设备验证。
- Data Card 的相册保存和分享需要在真实设备上再做权限和系统分享链路验证。
- 当前缺少单元测试，回归主要依赖 `make preflight` 和人工 UAT。

---

## 后端判断

当前阶段不需要后端。v1 的目标是验证本地个人工具是否能跑通“准备 -> 执行 -> 记录 -> 汇报”的闭环。

后续需要后端的触发条件：

- 用户需要账号登录和多设备同步。
- 用户需要云备份或换机恢复。
- 实验室需要共享 Protocol、库存或实验记录。
- Pro 订阅需要服务端权益校验。
- AI 功能需要服务端代理、队列、成本控制或审计。
- 数据卡片、Protocol 或模板需要社区分发。

在这些条件出现前，继续保持本地优先更合适。
